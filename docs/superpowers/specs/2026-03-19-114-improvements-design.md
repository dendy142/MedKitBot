## Волна 1 — Polish & UX

Цель: текущий функционал ощущается как готовый продукт.

### Навигация и отзывчивость

**#1 Хлебные крошки в заголовках**
Каждый экран показывает путь: `🏠 → Аптечка "Дом" → Ибупрофен`. Пользователь всегда знает где он. Реализация: утилита `breadcrumb(parts)` в `format.js`, вызывается при формировании текста каждого экрана.

**#2 Кнопка «Назад» везде**
Унифицированная навигация: карточка лекарства → список аптечки → список аптечек → главное меню. Добавить кнопку `◀️ Назад` в каждую клавиатуру. Callback формат: `back:target` (например `back:medkit:uuid`, `back:medkits`, `back:menu`).

**#3 Быстрые действия из главного меню**
Кнопки на главном экране: «⏰ Приёмы сегодня», «➕ Добавить лекарство», «🛒 Покупки». Без погружения в аптечку. При нажатии «➕» — выбор аптечки, затем визард.

**#4 Подтверждение деструктивных действий**
Удаление аптечки, лекарства, очистка списка покупок — двойное подтверждение. Первое нажатие: «Вы уверены? Это действие нельзя отменить.» + кнопки «Да, удалить» / «Отмена». Callback: `confirm:delete:entity:id`.

**#5 Индикация загрузки**
При длительных операциях (экспорт, импорт, массовые действия): `answerCallbackQuery({ text: '⏳ Загружаю...' })` немедленно, затем результат. Кнопки не должны казаться залипшими.

### Тексты и форматирование

**#6 Единый стиль эмодзи**
Ревизия всех сообщений. Маппинг в `config.js`:
```js
EMOJI: {
  categories: { painkillers: '💊', antibiotics: '🦠', vitamins: '🌿', ... },
  status: { expired: '🔴', expiring: '🟡', low: '🟠', ok: '🟢', favorite: '⭐' },
  actions: { add: '➕', edit: '✏️', delete: '🗑', back: '◀️', next: '▶️' },
  nav: { home: '🏠', medkit: '🧳', medicine: '💊', schedule: '⏰', shopping: '🛒', stats: '📊' }
}
```

**#7 Пустые состояния**
Каждый список при отсутствии данных показывает дружелюбное сообщение с подсказкой:
- Нет аптечек: «У вас пока нет аптечек. Создайте первую — нажмите ➕»
- Нет лекарств: «Аптечка пуста. Добавьте первое лекарство 💊»
- Нет приёмов: «Сегодня нет запланированных приёмов. Создайте расписание для ваших лекарств ⏰»
- Нет покупок: «Список покупок пуст 🎉»

**#8 Склонение количества**
Утилита `pluralize(n, one, few, many)` в `format.js`:
- `pluralize(1, 'таблетка', 'таблетки', 'таблеток')` → «1 таблетка»
- `pluralize(3, ...)` → «3 таблетки»
- `pluralize(5, ...)` → «5 таблеток»
Применить ко всем числительным в интерфейсе.

**#9 Относительные даты**
В списках: «Истекает через 3 дня», «Истёк 2 недели назад». Утилита `relativeDate(date)` в `format.js`. На карточке — точная дата. Пороги: <24ч → «сегодня/завтра», <7д → «через N дней», <30д → «через N недель», иначе точная дата.

**#10 Сокращение длинных названий**
Утилита `truncate(text, maxLen = 25)` в `format.js`. В кнопках и списках: обрезать с «...». Полное имя — только на карточке лекарства.

### Визарды и ввод

**#11 Пропуск необязательных шагов**
В визарде добавления лекарства шаги теги, фото, заметки — кнопка «Пропустить ▶️» единообразно на каждом необязательном шаге. Обязательные: имя, количество. Остальные — опциональны.

**#12 Предпросмотр перед сохранением**
Финальный шаг визарда — полная карточка лекарства с данными:
```
📋 Предпросмотр
━━━━━━━━━━━━━━━
💊 Ибупрофен 400мг
📁 Обезболивающие
📅 Годен до: 15.03.2027
📦 Количество: 30 таблеток
🏷 Теги: от головы, НПВС
📝 Заметки: После еды

[✅ Сохранить] [✏️ Редактировать] [❌ Отменить]
```

**#13 Отмена визарда**
Кнопка «❌ Отмена» на каждом шаге визарда. Если заполнены данные — подтверждение: «Данные будут потеряны. Отменить?». Если ничего не заполнено — отмена без вопросов.

**#14 Валидация ввода с подсказками**
При неверном формате — не «Ошибка», а пример:
- Дата: «Введите дату в формате ДД.ММ.ГГГГ, например 15.03.2027»
- Количество: «Введите число, например 30 или 0.5»
- Имя: «Введите название лекарства (до 100 символов)»

**#15 Запоминание последней категории**
При добавлении нескольких лекарств подряд — последняя выбранная категория показывается первой в списке с пометкой «(последняя)». Хранить в `sessions` как `lastCategory:{userId}`.

### Карточка лекарства

**#16 Прогресс-бар количества**
Визуальный бар на карточке:
```
📦 Количество: 24/30
[████████░░] 80% 🟢
```
Цвета: >50% → 🟢, 20-50% → 🟡, <20% → 🔴. Утилита `progressBar(current, max, width = 10)` в `format.js`.

**#17 Статус-бейджи**
Компактные бейджи в заголовке карточки: `🔴 Истёк` / `🟡 Истекает через 5д` / `🟠 Мало` / `⭐ Избранное`. Несколько бейджей одновременно: `💊 Ибупрофен ⭐🟡`.

**#18 Быстрый restock**
Кнопки прямо на карточке: `[+1] [+5] [+10] [Ввести число]`. Callback: `med:uuid:restock:1`, `med:uuid:restock:5`, etc. Без входа в отдельный режим.

**#19 Связанные расписания на карточке**
Секция на карточке лекарства:
```
⏰ Расписания:
  • Ежедневно в 08:00 (1 таблетка) ▶️
  • Ежедневно в 20:00 (1 таблетка) ▶️
[➕ Добавить расписание]
```

### Списки

**#20 Счётчики в кнопках аптечек**
Название кнопки: `🧳 Домашняя (12 💊)`. Количество лекарств (не архивных) через COUNT запрос.

**#21 Индикаторы проблем в аптечках**
Если есть проблемы: `🧳 Домашняя (12 💊, ⚠️ 3)`. Число проблем = просроченные + истекающие + на исходе.

**#22 Групповые операции**
Режим множественного выбора: кнопка «☑️ Выбрать несколько» → чекбоксы у лекарств → действия: «📦 Переместить», «🗄 Архивировать», «🗑 Удалить». Callback: `multiselect:med:uuid:toggle`, хранить выбранные в сессии.

**#23 Сортировка запоминается**
Выбранная сортировка сохраняется в `users.settings.defaultSort`. Применяется автоматически при открытии списка. Можно менять в любой момент.

### Мелочи

**#24 Тактильный отклик**
При подтверждении приёма: `answerCallbackQuery({ text: '✅ Принято!' })`. При пропуске: `answerCallbackQuery({ text: '⏭ Пропущено' })`. Краткое подтверждение действия.

**#25 Дата добавления на карточке**
Внизу карточки: `📅 Добавлено: 15.01.2026`. Контекст для пользователя — как давно лекарство в аптечке.

## Волна 2 — Умные фичи

Цель: бот думает за пользователя, используя только собственные данные. Никаких внешних сервисов.

### Умные предупреждения

**#27 Предупреждение "Пора докупить"**
Если лекарство с расписанием падает ниже настраиваемого порога (lowStockCount) — уведомление: «⚠️ Ибупрофен заканчивается (осталось 3). Добавить в список покупок?» + инлайн-кнопка `[🛒 Добавить]` / `[Позже]`. Проверка в cron `expiry-check.js`.

**#28 Автодобавление в shopping list**
Настройка в `settings.autoShoppingList: true/false` (по умолчанию false). Если включено — при падении ниже порога лекарство автоматически попадает в список покупок. Дедупликация: не добавлять если уже есть.

**#29 Конфликт расписаний**
При создании расписания — проверка: есть ли другое расписание на это же лекарство с тем же `time_value`. Если да: «⚠️ У вас уже есть приём Ибупрофена в 08:00. Создать ещё одно?» + `[Да]` / `[Отмена]`.

**#30 Дубликат лекарства**
При добавлении — проверка по `name.toLowerCase().trim()` в той же аптечке. Если найдено совпадение: «💊 В этой аптечке уже есть "Ибупрофен 200мг". Добавить новое или перейти к существующему?» + `[Добавить новое]` / `[Перейти]`.

### Умные подсказки

**#31 Подсказка дозировки из истории**
Если пользователь вводит имя лекарства и в его данных (любая аптечка) уже есть лекарство с похожим именем — предложить параметры: «Вы раньше добавляли "Ибупрофен 400мг" (Обезболивающие). Использовать те же параметры?» + `[Да, использовать]` / `[Нет, ввести вручную]`. Поиск по `toLowerCase().includes()`.

**#32 Шаблоны лекарств**
Функция «Добавить из моих лекарств» — список уникальных лекарств пользователя (из всех аптечек). Выбор → все поля копируются (кроме количества и срока годности). Query: SELECT DISTINCT ON (name) из medicines пользователя.

**#33 Авто-категория по ключевым словам**
Словарь в `config.js`:
```js
CATEGORY_KEYWORDS: {
  'Обезболивающие': ['ибупрофен', 'нурофен', 'парацетамол', 'кетанов', 'анальгин', 'аспирин', 'цитрамон'],
  'Антибиотики': ['амоксициллин', 'азитромицин', 'цефтриаксон', 'ципрофлоксацин'],
  'Витамины': ['витамин', 'омега', 'магний', 'кальций', 'цинк', 'железо', 'йод'],
  'Желудочные': ['омепразол', 'фосфалюгель', 'смекта', 'мезим', 'панкреатин', 'линекс'],
  'Антигистаминные': ['цетиризин', 'лоратадин', 'супрастин', 'зиртек', 'кларитин'],
  'Сердечные': ['валидол', 'корвалол', 'нитроглицерин', 'каптоприл', 'амлодипин'],
  'Жаропонижающие': ['нурофен детский', 'панадол', 'ибуклин'],
  'Противовирусные': ['арбидол', 'тамифлю', 'ингавирин', 'кагоцел'],
  'Антисептики': ['мирамистин', 'хлоргексидин', 'перекись', 'йод', 'зелёнка'],
}
```
При вводе имени — поиск по словарю, автовыбор категории с возможностью изменить.

**#34 Подсказка при дате в прошлом**
Если пользователь вводит срок годности, который уже прошёл: «⚠️ Эта дата уже прошла. Лекарство будет отмечено как просроченное. Продолжить?» + `[Да]` / `[Ввести другую дату]`.

### Умная статистика

**#35 Тренд приверженности**
Не просто «80% за неделю», а сравнение: «📊 Эта неделя: 80% (↑ с 65%)». Расчёт: текущая неделя vs предыдущая. Стрелки: ↑ улучшение, ↓ ухудшение, → без изменений.

**#36 Худшее время дня**
Анализ пропусков за 30 дней. Группировка по `time_value` расписания. «⚠️ Вы чаще всего пропускаете вечерний приём (19:00) — 40% пропусков». Показывать в недельной/месячной статистике.

**#37 Серия приёмов (streak)**
«🔥 12 дней подряд без пропусков!» Расчёт: count consecutive days where all intake_logs.status = 'taken'. Показывать на дашборде и при подтверждении приёма. Сброс при первом пропуске.

**#38 Статистика по лекарству**
На карточке лекарства (если есть расписание):
```
📊 Статистика:
  Принято: 45 из 52 (86%)
  Текущая серия: 6 дней
  Лучшая серия: 14 дней
```

### Умное расписание

**#39 Предложение расписания после добавления**
После сохранения лекарства: «⏰ Хотите создать расписание приёма?» + `[Да]` / `[Нет, позже]`. Если «Да» — переход в визард создания расписания для этого лекарства.

**#40 Автопауза при нуле**
Когда quantity = 0 и есть активные расписания: «💊 Ибупрофен закончился. Расписание поставлено на паузу.» Автоматическая пауза + уведомление. Можно возобновить вручную.

**#41 Предложение возобновить при рестоке**
При пополнении лекарства — проверка: есть ли приостановленные расписания. Если да: «Вы пополнили Ибупрофен. Возобновить расписание?» + `[▶️ Возобновить]` / `[Нет]`.

### Умные уведомления

**#42 Тихие часы**
Настройка в `settings.quietHours: { from: '23:00', to: '07:00' }`. Уведомления, попадающие в этот диапазон, откладываются до окончания тихих часов. Не применяется к подтверждению приёма — только к автоматическим уведомлениям (cron).

**#43 Адаптивные напоминания**
Анализ: если пользователь подтверждает приём за >5 минут до напоминания (3+ раза подряд), предложить: «Вы обычно принимаете лекарство раньше. Сдвинуть напоминание?» Раз в 30 дней, не навязчиво.

**#44 Группировка уведомлений**
Если в одну минуту >=2 напоминания — одно сообщение:
```
⏰ Пора принять лекарства:
  💊 Ибупрофен 400мг — 1 таблетка
  💊 Витамин D 1000МЕ — 1 капсула
  💊 Омега-3 1000мг — 1 капсула
[✅ Всё принято] [Подробнее]
```
Кнопка «Всё принято» — отмечает все разом.

**#45 Еженедельный отчёт**
Опциональный (settings.weeklyReport: true/false). Отправляется раз в неделю (понедельник утром): сводка приверженности за неделю, лекарства на исходе, просроченные, покупки. Cron-задача.

## Волна 3 — Семья и профили

Цель: бот для всей семьи.

### Профили членов семьи

**#46 Профили в аптечке**
Новая таблица `profiles`:
```sql
CREATE TABLE profiles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  name TEXT NOT NULL,
  age INTEGER,
  icon TEXT DEFAULT '👤',
  created_at TIMESTAMPTZ DEFAULT now()
);
```
Привязка к пользователю (не аптечке). Профиль — это «для кого лекарство». Один пользователь может иметь несколько профилей.

**#47 Привязка лекарства к профилю**
Новое поле `medicines.profile_id UUID REFERENCES profiles(id)`. NULL = общее лекарство. При добавлении/редактировании: «Для кого?» → список профилей + «Общее».

**#48 Привязка расписания к профилю**
Новое поле `schedules.profile_id UUID REFERENCES profiles(id)`. Напоминания содержат имя профиля: «⏰ Пора дать Мише: Нурофен 5мл».

**#49 Фильтр по профилю**
В списке лекарств аптечки: кнопка `[👤 Фильтр]` → выбор профиля → показ только его лекарств. Callback: `medkit:uuid:filter:profile:profileId`.

**#50 Статистика по профилю**
В разделе статистики: выбор профиля → приверженность, стрики, пропуски считаются отдельно.

**#51 Дашборд с профилями**
Если есть профили — главное меню расширяется:
```
👦 Миша: 2 приёма сегодня, ✅ 1 принято
👩 Мама: 3 приёма, ⚠️ 1 пропущен
📊 Общее: 12 💊, ⚠️ 2 истекают
```

**#52 Иконки профилей**
При создании/редактировании профиля — выбор иконки: `👶 👦 👧 👨 👩 👴 👵 🐱 🐶 🐰`. Инлайн-клавиатура 5×2.

### Управление профилями

**#53 Создание профиля**
Визард: имя (текст) → возраст (число, опционально) → иконка (выбор). Минимум 2 шага (имя обязательно). Вход: главное меню → настройки → профили → «Добавить профиль».

**#54 Редактирование профиля**
Карточка профиля с кнопками: `[✏️ Имя]` `[🔢 Возраст]` `[😊 Иконка]` `[🗑 Удалить]`.

**#55 Удаление профиля**
С подтверждением и выбором: «Перенести лекарства профиля в "Общие" или удалить вместе с профилем?» + `[Перенести]` / `[Удалить всё]` / `[Отмена]`.

**#56 Профиль по умолчанию**
Настройка `settings.defaultProfileId`. При добавлении лекарства — автоматически привязывает к выбранному профилю (можно изменить на шаге выбора).

### Здоровье и заметки

**#57 Заметки к приёму**
При подтверждении приёма — дополнительная кнопка `[📝 Заметка]`. При нажатии — текстовый ввод, сохраняется в `intake_logs.notes`. Опционально, не блокирует подтверждение.

**#58 Причина пропуска**
При «Пропущено» — выбор причины: `[Забыл]` `[Плохое самочувствие]` `[Закончилось]` `[Решение врача]` `[Другое]`. Хранится в `intake_logs.skip_reason`. При «Другое» — текстовый ввод.

**#59 Журнал самочувствия**
Новая таблица `wellbeing_logs`:
```sql
CREATE TABLE wellbeing_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
```

## Волна 4 — Надёжность

Цель: бот не ломается, не теряет данные.

### Обработка ошибок

**#64 Глобальный error handler**
```js
bot.catch((err) => {
  console.error({ timestamp: new Date().toISOString(), error: err.message, stack: err.stack, userId: err.ctx?.from?.id });
  err.ctx?.reply('⚠️ Произошла ошибка. Попробуйте ещё раз или вернитесь в /menu').catch(() => {});
});
```
Пользователь никогда не видит необработанную ошибку.

**#65 Устаревшие callback**
Перед каждым действием по callback — проверка существования сущности. Если не найдена:
```
answerCallbackQuery({ text: '⚠️ Данные устарели. Вернитесь в /menu', show_alert: true })
```
Утилита `ensureExists(entity, ctx)` — возвращает false и отвечает пользователю если сущность удалена.

**#66 Таймаут сессий**
В cron-задаче (или при каждом обращении пользователя) — проверка: если сессия `updated_at` > 24 часов назад — удалить. При попытке продолжить устаревший визард: «Ваш предыдущий ввод устарел. Начнём заново?» + `[Начать]` / `[Отмена]`.

**#67 Retry при ошибках Supabase**
Обёртка `withRetry(fn, retries = 1, delay = 500)`:
```js
async function withRetry(fn, retries = 1, delay = 500) {
  try { return await fn(); }
  catch (err) {
    if (retries > 0) { await sleep(delay); return withRetry(fn, retries - 1, delay); }
    throw err;
  }
}
```
Применить к критическим запросам (intake confirmation, medicine creation).

**#68 Обработка rate limit Telegram**
В `bot.catch()` проверять `err.error_code === 429`. Если да — `Retry-After` header → ждать и повторять. Важно для cron-задач, рассылающих массовые уведомления.

### Валидация данных

**#69 Санитизация текстового ввода**
Утилита `sanitize(text, maxLen)` в `format.js`:
- `text.trim()`
- Удаление управляющих символов (кроме \n)
- Лимит: имя — 100, заметки — 500, теги — 50 на тег
- Возврат `null` если после санитизации пусто

**#70 Валидация даты**
- Не дальше 10 лет в будущее (2036)
- При дате >5 лет — предупреждение: «Дата очень далёкая. Вы уверены?»
- Парсинг нескольких форматов: ДД.ММ.ГГГГ, ДД/ММ/ГГГГ, ГГГГ-ММ-ДД

**#71 Валидация количества**
- Только положительные числа
- Максимум 99999
- Дробные до 1 знака (0.5)
- Запятая и точка как разделитель: «0,5» → 0.5

**#72 Дедупликация callback**
Middleware: сохранять `callback_query.id` в Set (в памяти, живёт до следующего вызова функции). Если повторный — `answerCallbackQuery()` без действия.

**#73 Проверка прав**
Middleware `checkRole(minRole)`:
```js
// 'viewer' < 'editor' < 'owner'
function checkRole(medkitId, minRole) {
  return async (ctx, next) => {
    const member = await getMember(medkitId, ctx.dbUser.id);
    if (roleLevel(member.role) < roleLevel(minRole)) {
      return ctx.answerCallbackQuery({ text: '🔒 Недостаточно прав', show_alert: true });
    }
    return next();
  };
}
```
Применить ко всем действиям редактирования/удаления.

## Cron-задания

Все крон-задания вызываются через cron-job.org → HTTP GET/POST к Vercel endpoints.

### Существующие (обновлённые)

| Endpoint | Частота | Описание |
|----------|---------|----------|
| `POST /api/cron/reminders` | Каждые 5 мин | Генерация intake_logs + отправка напоминаний (с группировкой #44, тихими часами #42, идемпотентностью #76) |
| `POST /api/cron/expiry-check` | Ежедневно 06:00 UTC | Проверка сроков + low stock предупреждения #27 + автодобавление в покупки #28 |
| `POST /api/cron/digest` | Каждый час | Ежедневный дайджест + подсказки новичкам #83 |

### Новые

| Endpoint | Частота | Волна | Описание |
|----------|---------|-------|----------|
| `POST /api/cron/weekly-report` | По понедельникам 08:00 UTC | 2 | Еженедельный отчёт приверженности #45 |
| `POST /api/cron/wellbeing` | Ежедневно 19:00 UTC | 3 | Опрос самочувствия #59 |
| `POST /api/cron/session-cleanup` | Ежедневно 03:00 UTC | 4 | Очистка устаревших сессий #66 |
| `POST /api/cron/inactive-users` | Еженедельно (воскресенье) | 5 | Напоминание неактивным #89 |

Все endpoints защищены `CRON_SECRET` в заголовке `Authorization: Bearer <CRON_SECRET>`.

### Настройка в cron-job.org

Для каждого задания в cron-job.org:
- URL: `https://<vercel-domain>/api/cron/<name>`
- Method: POST
- Header: `Authorization: Bearer <CRON_SECRET>`
- Header: `Content-Type: application/json`
