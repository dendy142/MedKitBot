# Medkit Bot — Полная спецификация

## 1. Общее описание

| Параметр | Значение |
|---|---|
| **Telegram** | @my_med_kit_bot |
| **Тип** | Классический Telegram-бот (Bot API, не TWA) |
| **Язык** | Русский |
| **Среда** | Только личные сообщения (private chat) |
| **Стек** | Node.js + grammY + Supabase PostgreSQL + Vercel (serverless webhook) |
| **Навигация** | Полностью Inline Keyboard (никаких Reply Keyboard) |
| **Команды BotFather** | /start, /help, /cancel |

---

## 2. Основные сущности

1. **Пользователь** — Telegram-аккаунт, часовой пояс, настройки уведомлений и отображения
2. **Аптечка** — именованная коллекция лекарств (несколько на пользователя: «Домашняя», «Дачная», «В дорогу»...). Бывает личная и общая
3. **Лекарство** — название, дозировка, категория, теги, срок годности, количество+единица, до 5 фото, заметки. Поддерживает soft-delete (архив)
4. **Расписание приёма (курс)** — привязка лекарства к времени/периоду; у одного лекарства может быть несколько активных курсов одновременно. Поддерживает паузу и повтор
5. **Запись приёма** — факт приёма (дата/время, статус, примечание пользователя)
6. **Приглашение** — deep link или @username для совместного доступа к аптечке
7. **Список покупок** — ручной список лекарств к покупке, общий + по аптечкам
8. **Лог действий** — аудит-лог в БД для дебага и будущей админки

---

## 3. Категории лекарств

### Предустановленные
- Обезболивающие
- Жаропонижающие
- Антибиотики
- Антигистаминные
- Витамины и БАДы
- Желудочно-кишечные
- Сердечно-сосудистые
- Наружные средства (мази, пластыри)
- Прочее

Пользователь может создавать **свои категории**.

---

## 4. Единицы измерения количества

Расширенный набор:
- Таблетки
- Капсулы
- Миллилитры (мл)
- Капли
- Ампулы
- Пакетики
- Штуки (пластыри, свечи и т.д.)

---

## 5. Теги/метки лекарств

Пользовательские теги: «для детей», «рецептурное», «аллергия», произвольный текст.
- Лекарство может иметь **несколько тегов**
- Теги отображаются в карточке лекарства
- Доступны для **фильтрации** в списке аптечки

## 6. Навигация и UX

### 6.1. Принципы

- **Inline-кнопки** — вся навигация через InlineKeyboard, никаких Reply Keyboard
- **Редактирование сообщения** — при навигации бот **редактирует** текущее сообщение (не спамит новыми)
- **Кнопка «◀️ Назад»** на каждом экране
- **/cancel** + кнопка «❌ Отмена» — отмена любого пошагового диалога
- **Подтверждение удаления** — всегда переспрос «Точно удалить X?» с кнопками Да/Нет

### 6.2. Пагинация

- **8 элементов** на страницу
- Кнопки ◀️ / ▶️ для навигации
- Формат: `◀️ 1/3 ▶️`

### 6.3. Сортировка

Варианты сортировки списка лекарств:
- По имени (А-Я)
- По сроку годности (ближайший первый)
- По категории
- По остатку (наименьший первый)
- Приоритет проблем (просроченные → заканчивающиеся → остальные)

Сортировка по умолчанию настраивается в настройках пользователя.

### 6.4. Формат дат

**Ввод:** поддерживаются форматы:
- ММ.ГГГГ (03.2027)
- ДД.ММ.ГГГГ (15.03.2027)
- Inline-календарь (кнопки для выбора)

**Отображение:** настраивается в настройках пользователя.

### 6.5. Fallback на поиск

Если пользователь пишет произвольный текст (не в контексте диалога):
1. Бот пытается найти лекарство по названию
2. Если нашёл — показывает результаты
3. Если не нашёл — показывает главное меню с подсказкой «Не понял. Вот что я умею:»

## 7. Экраны и пользовательские сценарии

### 7.1. `/start` — Онбординг (интерактивный)

**Новый пользователь:**
1. Приветственное сообщение с кратким описанием возможностей бота
2. Выбор часового пояса (популярные города РФ: Москва, Санкт-Петербург, Екатеринбург, Новосибирск, Владивосток + «Другой»)
3. Автоматическое создание первой аптечки «Домашняя»
4. Бот предлагает **сразу добавить первое лекарство** и объясняет каждый шаг по ходу
5. После добавления → переход в главное меню

**Существующий пользователь:** `/start` → сразу главное меню.

- Количество позиций в списке покупок

### 7.4. Экран аптечки (подробный формат)

```
📦 Домашняя аптечка (12 лекарств)

✅ Парацетамол 500мг
   Остаток: 18 табл. | До: 03.2027
⚠️ Ибупрофен 200мг
   Остаток: 3 табл. | До: 06.2026
❌ Аспирин 100мг
   Остаток: 10 табл. | ПРОСРОЧЕНО
📉 Витамин D 1000МЕ
   Остаток: 2 капс. | До: 12.2027

[Парацетамол][Ибупрофен]
[Аспирин][Витамин D]
◀️ 1/2 ▶️

[+ Добавить][🔀 Сортировка][📂 Фильтр]
[👥 Поделиться][✏️ Редакт.][🗑 Удалить]
[📁 Архив][◀️ Назад]
```

**Индикаторы статуса:**
- ✅ — всё в порядке
- ⚠️ — скоро истекает срок годности
- ❌ — просрочено
- 📉 — заканчивается (низкий остаток)

**Действия:**
- Добавить лекарство
- Сортировка (по имени / по сроку / по категории / по остатку / приоритет проблем)
- Фильтр (по категории или тегу)
- Поделиться (шеринг)
- Редактировать название аптечки
- Удалить аптечку (с подтверждением)
- Архив (просмотр архивированных лекарств)

### 7.5. Добавление лекарства (пошаговый диалог)

**Шаг 1 — Название:**
```
📝 Введите название лекарства:

[❌ Отмена]
```
Пользователь вводит текст.

**Шаг 2 — Дозировка:**
```
💊 Введите дозировку (например: 500мг, 10мл, 200мкг):

[◀️ Назад][❌ Отмена]
```

**Шаг 3 — Категория:**
```
🏷 Выберите категорию:

[Обезболивающие][Жаропонижающие]
[Антибиотики][Антигистаминные]
[Витамины и БАДы][ЖКТ]
[Сердечно-сосуд.][Наружные]
[Прочее][+ Другая]
[◀️ Назад][❌ Отмена]
```

**Шаг 4 — Теги (мультивыбор):**
```
🔖 Выберите теги (можно несколько):

[✅ для детей][рецептурное]
[аллергия][+ Добавить тег]
[⏭ Пропустить]
[◀️ Назад][❌ Отмена]
```
Выбранные теги помечаются ✅. Можно добавить новый тег текстом.

**Шаг 5 — Срок годности:**
```
📅 Укажите срок годности:

Введите в формате ММ.ГГГГ или ДД.ММ.ГГГГ
или выберите в календаре:

[📅 Календарь]
[◀️ Назад][❌ Отмена]
```

**Шаг 6 — Количество:**
```
📏 Введите количество (число):

Затем выберите единицу измерения:
[Таблетки][Капсулы][мл]
[Капли][Ампулы][Пакетики][Штуки]
[◀️ Назад][❌ Отмена]
```

**Шаг 7 — Фото (опционально, до 5 штук):**
```
📷 Отправьте фото упаковки (до 5 штук):

Загружено: 1/5

[📷 Добавить ещё][✅ Готово]
[⏭ Пропустить]
[◀️ Назад][❌ Отмена]
```

**Шаг 8 — Заметки (опционально):**
```
📝 Добавьте заметки (например: «принимать после еды», «хранить в холодильнике»):

[⏭ Пропустить]
[◀️ Назад][❌ Отмена]
```

**Шаг 9 — Подтверждение:**
```
✅ Проверьте данные:

💊 Парацетамол 500мг
🏷 Жаропонижающие
🔖 #для_детей
📅 Срок: 03.2027
📏 30 таблеток
📷 1 фото
📝 Принимать после еды

[✅ Сохранить][✏️ Изменить][❌ Отмена]
```

На **каждом шаге**: кнопка «◀️ Назад» + «❌ Отмена» + поддержка /cancel.

### 7.6. Карточка лекарства (полная)

```
💊 Парацетамол 500мг
🏷 Жаропонижающие | #для_детей
📦 Аптечка: Домашняя
📅 Срок: 03.2027 (осталось 365 дн.)
📏 Остаток: 18/30 табл. (60%)
📝 Принимать после еды

🔄 Курс 1: 1 табл. × 2р/день (08:00, 20:00)
   День 5/14 | 🔥 Стрик: 5 дней
🔄 Курс 2: 1 табл. × утро
   Бессрочно | 🔥 Стрик: 21 день

[✏️ Ред.][➕ Пополнить][📷 Фото]
[📆 Приём][📂 Копир./Перем.][⭐]
[📋 История][🗑 В архив]
[◀️ Назад]
```

**Содержание:**
- Название + дозировка
- Категория + теги
- Аптечка (к какой принадлежит)
- Срок годности с обратным отсчётом дней
- Остаток: текущий/начальный + процент
- Заметки
- Все активные курсы с прогрессом и стриком

**Кнопки действий:**
- ✏️ Редактировать (пополям)
- ➕ Пополнить остаток
- 📷 Показать фото (по кнопке, не сразу)
- 📆 Управление приёмом/курсами
- 📂 Копировать/переместить в другую аптечку
- ⭐ Избранное (toggle)
- 📋 История изменений
- 🗑 В архив (soft-delete, с подтверждением)

---

### 7.7. Редактирование лекарства (пополям)

```
✏️ Редактирование: Парацетамол 500мг

Что изменить?

[Название][Дозировка]
[Категория][Теги]
[Срок годн.][Количество]
[Заметки][Фото]
[◀️ Назад]
```

При выборе поля:
1. Бот показывает текущее значение
2. Пользователь вводит новое
3. Возврат в карточку

**Оптимистичная блокировка:** если данные изменились с момента открытия карточки (другой участник общей аптечки) — бот предупреждает: «⚠️ Данные были изменены. Обновить?»

```
➕ Пополнение: Парацетамол 500мг

Текущий остаток: 18 табл.

[◀️ Назад][❌ Отмена]
```

```
📂 Парацетамол 500мг

Выберите действие:

[📋 Копировать][📦 Переместить]

```

Затем:

### 7.10. Избранное

- Кнопка ⭐ в карточке лекарства — toggle (нажал = избранное, нажал ещё раз = убрал)
- Избранные лекарства отображаются **в топе** любого списка (перед остальными)
- В списке помечены ⭐

### 7.11. Настройка приёма (курсы)

У одного лекарства может быть **несколько активных курсов** одновременно (например: 1 табл. утром + 2 табл. вечером).

#### Создание курса

**Шаг 1 — Режим времени:**
```
⏰ Когда принимать?

[🕐 Точное время][🌅 Период дня]
[◀️ Назад][❌ Отмена]
```

- **Точное время:** ввод HH:MM (например 08:00)
- **Период дня:** выбор утро / день / вечер / ночь (время привязано к настройкам пользователя)

**Шаг 2 — Дозировка за приём:**
```
💊 Сколько принимать за раз?

Введите количество (например: 1, 2, 0.5):

[◀️ Назад][❌ Отмена]
```

**Шаг 3 — Частота:**
```
📅 Как часто принимать?

[Ежедневно][Через день]
[По дням недели]
[◀️ Назад][❌ Отмена]
```

Для «По дням недели» — мультивыбор: Пн Вт Ср Чт Пт Сб Вс.

**Шаг 4 — Длительность:**
```
⏱ Длительность курса:

[♾ Бессрочно][📅 N дней]
[📅 До даты]
[◀️ Назад][❌ Отмена]
```

**Шаг 5 — Подтверждение:**
```
✅ Курс приёма:

💊 Парацетамол 500мг
⏰ 08:00 и 20:00
💊 1 табл. за приём
📅 Ежедневно, 14 дней

[✅ Создать][✏️ Изменить][❌ Отмена]
```

#### Управление курсами

```
📆 Курсы приёма: Парацетамол 500мг

🔄 Курс 1 (активен)
   1 табл. × 2р/день (08:00, 20:00)
   День 5/14

🔄 Курс 2 (на паузе)
   1 табл. × утро
   Бессрочно

[+ Добавить курс]
[✏️ Курс 1][⏸ Пауза 1][🗑 Удалить 1]
[✏️ Курс 2][▶️ Возобн. 2][🗑 Удалить 2]
[◀️ Назад]
```

**Действия с курсом:**
- ⏸ Пауза / ▶️ Возобновить — временная остановка без удаления
- 🔄 Повторить курс — создать новый с теми же настройками
- ✏️ Редактировать
- 🗑 Удалить (с подтверждением)

### 7.12. Приём лекарств (сегодня)

```
💊 Приём на сегодня (15.03.2026)

🕗 08:00
  ✅ Парацетамол 500мг — 1 табл.
  ⏳ Витамин D 1000МЕ — 1 капс.

🕑 14:00
  ⏳ Ибупрофен 200мг — 2 табл.

🕗 20:00
  ⏳ Парацетамол 500мг — 1 табл.

[✅ Витамин D — Принял]
[✅ Ибупрофен — Принял]
[◀️ Назад]
```

- Группировка по времени
- Статусы: ⏳ ожидает / ✅ принято / ❌ пропущено
- Кнопка «Принял» рядом с каждым ожидающим приёмом
- **Автоматическое списание остатка** при подтверждении
- При приёме можно **добавить примечание** (опциональная кнопка «📝 + примечание»)

### 7.13. Напоминания о приёме (подробное)

Бот отправляет **новое сообщение** (не редактирует) в назначенное время:

```
⏰ Напоминание о приёме

💊 Парацетамол 500мг
📏 Доза: 1 табл.
📦 Остаток: 17 табл.
📅 День 5 из 14
📝 Принимать после еды

[✅ Принял][⏰ +15 мин][❌ Пропуск]
```

**Логика:**
- Заметка лекарства показывается в напоминании
- Кнопки: Принял / Напомнить через 15 мин / Пропустить
- При нажатии «+15 мин» — повторное напоминание через 15 минут
- **Макс. 2 повторных** напоминания, потом статус → «пропущено»
- При «Принял» — можно добавить примечание к приёму

---

### 7.14. Ситуация «остаток = 0»

Когда подошло время приёма, а остаток лекарства 0:

```
⚠️ Лекарство закончилось!

💊 Парацетамол 500мг — остаток: 0 табл.

[🛒 В список покупок][➕ Пополнить]
[⏸ Приостановить курс]
[◀️ Назад]
```

---

### 7.15. Окончание курса

Когда курс достиг указанной длительности:

```
🎉 Курс завершён!

💊 Парацетамол 500мг
📅 14 дней
🔥 Соблюдение: 12 из 14 (86%)

[🔄 Повторить курс][✅ Завершить]
```

---

### 7.16. Уведомления о сроках годности

- За **N дней** до истечения (N настраивается, по умолчанию 30) → предупреждение ⚠️
- **В день истечения** → уведомление ❌
- Можно **отключить** в настройках

```
⚠️ Скоро истекает срок годности!

💊 Ибупрофен 200мг
📦 Домашняя аптечка
📅 Срок: 04.2026 (осталось 28 дней)

[📦 Открыть аптечку][🛒 В список покупок]
```

---

### 7.17. Уведомления об остатках

Когда остаток падает ниже порога (настраиваемый, по умолчанию 5 шт / 20%):

```
📉 Заканчивается лекарство!

💊 Витамин D 1000МЕ
📦 Домашняя аптечка
📏 Остаток: 3/30 капс. (10%)

[➕ Пополнить][🛒 В список покупок]
```

### 7.19. Общие аптечки — шеринг

#### Приглашение

**По ссылке:**
1. Владелец нажимает «👥 Поделиться» в экране аптечки
2. Выбирает роль для приглашённого: «Редактор» или «Только просмотр»
3. Бот генерирует deep link: `t.me/my_med_kit_bot?start=invite_XXXX`
4. Другой пользователь переходит по ссылке → присоединяется

**По @username:**
1. Владелец нажимает «👥 Поделиться» → «По юзернейму»
2. Вводит @username
3. Выбирает роль
4. Бот отправляет приглашение этому пользователю
5. Тот подтверждает кнопкой

#### Роли

| Роль | Просмотр | Добавление | Редактирование | Удаление | Управление участниками |
|---|---|---|---|---|---|
| **Владелец** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Редактор** | ✅ | ✅ | ✅ | ❌ | ❌ |
| **Только просмотр** | ✅ | ❌ | ❌ | ❌ | ❌ |

#### Управление участниками

```
👥 Участники: Семейная аптечка

👑 @username1 — Владелец
✏️ @username2 — Редактор
👁 @username3 — Только просмотр

[✏️ @username2 — Изменить роль]
[✏️ @username3 — Изменить роль]
[🗑 Удалить участника]
[🔄 Передать владение]
[📨 Пригласить]
[◀️ Назад]
```

**Действия:**
- Список участников с ролями
- Изменение роли участника (владелец)
- Удаление участника (владелец)
- Передача владения другому участнику
- Участник может **сам выйти** из аптечки

#### Уведомления об изменениях

При любом изменении в общей аптечке (добавление/удаление/редактирование лекарства) — **уведомление всем участникам**:

```
📢 Изменение в общей аптечке

📦 Семейная аптечка
👤 @username2 добавил:
💊 Нурофен 400мг

[📦 Открыть аптечку]
```

Можно **отключить** в настройках.

## 10. Cron-задачи (Vercel Cron Jobs)

### `api/cron/reminders.js`
- **Частота:** каждую минуту
- **Логика:**
  1. Найти все intake_logs с status = pending и planned_at ≤ now
  2. Отправить напоминания через Bot API
  3. Обработать snooze (повторные напоминания)
  4. Пометить как snoozed/skipped при превышении лимита

### `api/cron/expiry-check.js`
- **Частота:** раз в день (утром)
- **Логика:**
  1. Найти лекарства с expiry_date ≤ now + порог пользователя
  2. Отправить уведомления (если включены)
  3. Не дублировать уведомления (проверять что уже отправлено)

### `api/cron/digest.js`
- **Частота:** каждую минуту
- **Логика:**
  1. Найти пользователей, у которых время дайджеста ≤ now и ещё не отправлен сегодня
  2. Сформировать дайджест по настройкам пользователя
  3. Отправить
