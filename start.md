## 1. Курсы приёма (полностью отсутствует)

Таблицы `schedules` и `intake_logs` в БД есть, запросы в `src/db/queries/` написаны, но хендлеры не реализованы.

### 1.1. Создание курса (из карточки лекарства → кнопка «📆 Приём»)

Сейчас кнопка «📆 Приём» показывает заглушку «Скоро! Функция в разработке.»

**Шаг 1 — Режим времени:**
```
⏰ Когда принимать?

[🕐 Точное время][🌅 Период дня]
[◀️ Назад][❌ Отмена]
```

- **Точное время:** ввод HH:MM (например 08:00)
- **Период дня:** выбор утро / день / вечер / ночь (время привязано к настройкам пользователя)

**Шаг 2 — Дозировка за приём:**
```
💊 Сколько принимать за раз?

Введите количество (например: 1, 2, 0.5):

[◀️ Назад][❌ Отмена]
```

**Шаг 3 — Частота:**
```
📅 Как часто принимать?

[Ежедневно][Через день]
[По дням недели]
[◀️ Назад][❌ Отмена]
```

Для «По дням недели» — мультивыбор: Пн Вт Ср Чт Пт Сб Вс.

**Шаг 4 — Длительность:**
```
⏱ Длительность курса:

[♾ Бессрочно][📅 N дней]
[📅 До даты]
[◀️ Назад][❌ Отмена]
```

**Шаг 5 — Подтверждение:**
```
✅ Курс приёма:

💊 Парацетамол 500мг
⏰ 08:00 и 20:00
💊 1 табл. за приём
📅 Ежедневно, 14 дней

[✅ Создать][✏️ Изменить][❌ Отмена]
```

### 1.2. Управление курсами

```
📆 Курсы приёма: Парацетамол 500мг

🔄 Курс 1 (активен)
   1 табл. × 2р/день (08:00, 20:00)
   День 5/14

🔄 Курс 2 (на паузе)
   1 табл. × утро
   Бессрочно

[+ Добавить курс]
[✏️ Курс 1][⏸ Пауза 1][🗑 Удалить 1]
[✏️ Курс 2][▶️ Возобн. 2][🗑 Удалить 2]
[◀️ Назад]
```

**Действия с курсом:**
- ⏸ Пауза / ▶️ Возобновить — временная остановка без удаления
- 🔄 Повторить курс — создать новый с теми же настройками
- ✏️ Редактировать
- 🗑 Удалить (с подтверждением)

### 1.3. Отображение курсов в карточке лекарства

В карточке лекарства нужно показывать все активные курсы с прогрессом и стриком:
```
🔄 Курс 1: 1 табл. × 2р/день (08:00, 20:00)
   День 5/14 | 🔥 Стрик: 5 дней
```

## 3. Напоминания о приёме (полностью отсутствует)

Крон `api/cron/reminders.js` отсутствует.

Бот отправляет **новое сообщение** (не редактирует) в назначенное время:

```
⏰ Напоминание о приёме

💊 Парацетамол 500мг
📏 Доза: 1 табл.
📦 Остаток: 17 табл.
📅 День 5 из 14
📝 Принимать после еды

[✅ Принял][⏰ +15 мин][❌ Пропуск]
```

**Логика:**
- Заметка лекарства показывается в напоминании
- Кнопки: Принял / Напомнить через 15 мин / Пропустить
- При нажатии «+15 мин» — повторное напоминание через 15 минут
- **Макс. 2 повторных** напоминания, потом статус → «пропущено»
- При «Принял» — можно добавить примечание к приёму

### Крон `api/cron/reminders.js`
- **Частота:** каждую минуту
- **Логика:**
  1. Найти все intake_logs с status = pending и planned_at ≤ now
  2. Отправить напоминания через Bot API
  3. Обработать snooze (повторные напоминания)
  4. Пометить как snoozed/skipped при превышении лимита

## 7. Общие аптечки — шеринг (полностью отсутствует)

Кнопка «👥 Поделиться» в аптечке — заглушка. Таблица `invitations` есть, запросы написаны.

### Приглашение

**По ссылке:**
1. Владелец нажимает «👥 Поделиться» в экране аптечки
2. Выбирает роль для приглашённого: «Редактор» или «Только просмотр»
3. Бот генерирует deep link: `t.me/my_med_kit_bot?start=invite_XXXX`
4. Другой пользователь переходит по ссылке → присоединяется

**По @username:**
1. Владелец нажимает «👥 Поделиться» → «По юзернейму»
2. Вводит @username
3. Выбирает роль
4. Бот отправляет приглашение этому пользователю
5. Тот подтверждает кнопкой

### Роли

| Роль | Просмотр | Добавление | Редактирование | Удаление | Управление участниками |
|---|---|---|---|---|---|
| **Владелец** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Редактор** | ✅ | ✅ | ✅ | ❌ | ❌ |
| **Только просмотр** | ✅ | ❌ | ❌ | ❌ | ❌ |

### Управление участниками

```
👥 Участники: Семейная аптечка

👑 @username1 — Владелец
✏️ @username2 — Редактор
👁 @username3 — Только просмотр

[✏️ @username2 — Изменить роль]
[✏️ @username3 — Изменить роль]
[🗑 Удалить участника]
[🔄 Передать владение]
[📨 Пригласить]
[◀️ Назад]
```

**Действия:**
- Список участников с ролями
- Изменение роли участника (владелец)
- Удаление участника (владелец)
- Передача владения другому участнику
- Участник может **сам выйти** из аптечки

### Уведомления об изменениях

При любом изменении в общей аптечке — **уведомление всем участникам**:

```
📢 Изменение в общей аптечке

📦 Семейная аптечка
👤 @username2 добавил:
💊 Нурофен 400мг

[📦 Открыть аптечку]
```

Можно **отключить** в настройках.

## Справка: таблицы БД (уже существуют, но не интегрированы)

### `schedules`
| Поле | Тип | Описание |
|---|---|---|
| id | uuid (PK) | Уникальный ID |
| medicine_id | uuid (FK → medicines) | Лекарство |
| user_id | uuid (FK → users) | Пользователь |
| time_mode | text | exact / period |
| time_value | text | HH:MM или morning/afternoon/evening/night |
| dose_per_intake | numeric | Дозировка за приём |
| frequency | text | daily / every_other_day / weekly_days |
| frequency_days | integer[] | Дни недели (0=Пн...6=Вс) |
| duration_type | text | indefinite / days / until_date |
| duration_value | text | Кол-во дней или дата |
| start_date | date | Дата начала курса |
| status | text | active / paused / completed |
| created_at | timestamptz | Дата создания |

### `intake_logs`
| Поле | Тип | Описание |
|---|---|---|
| id | uuid (PK) | Уникальный ID |
| schedule_id | uuid (FK → schedules) | Расписание |
| medicine_id | uuid (FK → medicines) | Лекарство |
| user_id | uuid (FK → users) | Пользователь |
| planned_at | timestamptz | Запланированное время |
| status | text | pending / taken / skipped / snoozed |
| confirmed_at | timestamptz | Фактическое время приёма |

### `invitations`
